# 十八、图形与Canvas

# 减少重绘

  • requestAnimationFrame(fn?:Function):Long

并不是专用于 Canvas，而是用于浏览器重绘时阶段性的进行动画，在浏览器触发重绘事件时调用参数 fn，若在 fn 中继续进行动画，则可以减少浏览器进行重绘，requestAnimationFrame() 返回一个请求 id。

  • cancelAnimationFarme
    用于取消 requestAnimationFrame，传入 requestAnimationFrame 的 id。

# 画布与上下文

使用 html5 的新标签 <canvas> 进行绘画，属性 width 和 height 设置画布宽高。

注意，若直接在选择器上设置 css 样式，会导致图案比例不对。

<canvas style="border:1px solid red" width="200" height="200"></canvas>

获取 canvas 节点后使用 getContext(type:'2d'|...) 取得绘图上下文，即画笔，并利用上下文中作画。

注意，getContext 方法未必存在，平面图形传参 '2d'。

若绘画超过画布，不显示。

let canvas = document.getElementById('canvas')
if(canvas.getContext){
    let context = canvans.getContext('2d')
}
  • 画布转图片
    canvas 元素还有一个 toDataURL(MIMEtype:"image/png"|....) 方法，用于将画布转为图片链接。
let canvas = document.getElementById('canvas')
let imgURL = canvas.toDataURL('image/png')

# 画图

基于画笔上下文 context ，2d 画图默认起始坐标是左上角，定为 (0,0)，x 坐标向右增长，y 坐标向下增长。

# 填充和描边

皆为设置属性。 fillStyle 为填充属性，strokeStyle 为描边属性。

context.fillStyle = colorStr:'red'|'#FFF'
context.strokeStyle = colorStr:'red'|'#FFF'

# 绘制矩形

画布中只可以直接画矩形（rectangle），每一个绘制矩形的方法皆有四个参数，分别是x坐标、y坐标、矩形宽度、矩形长度，皆为数值，表示像素值。

  • fillRect(x,y,width,height)
    填充一个矩形
  • strokeRect(x,y,width,height) 描绘一个矩形轮廓
  • clearRect(x,y,width,height) 画一个白底矩形，类似于清除

# 绘制路径

调用上下文的 beginPath 方法表示开始绘制路径，可以描绘的路径如下：

  • arc(x,y,radius,sAngle,eAngle,resverse)
    从坐标(x,y)为圆心，从东的方向顺时针，以 radius 为半径，画一条弧线，参数 resverse 表示是否逆时针计算角度。注意 sAngle 和 eAngle 为弧度制，eAngle 若不合法，则默认为 2*Math.PI，sAngle不好说。
  • arcTo(x,y,x1,y1,radius)
    从起点开始，经过点(x,y)，到达(x1,y1)，弧线半径为 radius
  • quadraticCurveTo(cx, cy, x, y)
    以(cx, cy)为控制点，绘制一条从上一点到(x, y) 的弧线。
  • bezierCurveTo(c1x, c1y, c2x, c2y, x, y)
    以(c1x, c1y)和(c2x, c2y)为控制点， 绘制一条从上一点到(x, y)的弧线。
  • lineTo(x,y)
    从上一点到点(x,y)的直线
  • moveTo(x,y)
    把绘制光标移动到(x, y)。
  • rect(x, y, width, height)
    画矩形

# 路径完成

  • closePath()
    返回起始点
  • fill()
    填充路径
  • stroke()
    描绘路径

# 检查路径

检查路径的方法是检查某点是否如期的在路径上。

context.isPointInPath(x,y):Boolean

# 绘制文字

绘制文字前，需了解上下文的文字相关属性。

但是此属性有默认值，可以不设。

# 文本属性
  • font
    字体相关，例如字体大小，字体样式等。
  • textAlign
    字体对齐方式，有 start/end 和 right/left 可选，但是推荐使用 start/end，表示坐标在文本的左右侧。
  • textBaseLine
    文本的基线，值包括 "top" 、 "hanging" 、 "middle" 、 "alphabetic"、"ideographic"和"bottom"。
# 绘制文字

context.fillText(words, x, y) 和 context.strokeText(words, x, y);

# 变换

用户操作图像，即

  • rotate(angle)
    围绕原点把图像旋转 angle 弧度
  • scale(scaleX,scaleY)
    x轴缩放scaleX倍，y轴缩放scaleY轴
  • translate(x, y)
    把原点移动到(x, y)。执行这个操作后，坐标(0, 0)就会变成(x, y)。
  • transform(m1_1, m1_2, m2_1, m2_2, dx, dy) 通过矩阵乘法直接修改矩阵。

# 绘制图像

获取图像，调用上下文的 drawImage() 方法将图像绘制在画布中。

  • context.drawImage(img,x,y,width,height)
  • context.drawImage(img, x, y, width , height, c_x, c_y, c_width, c_height);

# 阴影

  • shadowColor 阴影颜色
  • shadowOffsetX 阴影相对于X轴的偏移量
  • shadowOffsetY 阴影相对于Y轴的偏移量
  • shadowBlur 阴影模糊量

# 渐变

调用上下文的 createLinearGradient(s_x,s_y,e_x,e_y) 方法创建一个渐变对象。

该对象有 addColorStop(num,color) 设置渐变色

其中 num 表示色标位置，范围为 0~1，0 表示第一种颜色，1 表示最后一种颜色。

可以把渐变对象赋给 fillStyle 或 strokeStyle 属性，从而以渐变填充 或描画绘制的图形：

let gradient = context.createLinearGradient(30, 30, 70, 70); 
gradient.addColorStop(0, "white"); 
gradient.addColorStop(1, "black");

# 图案

调用上下文的 createPatten(img,repeat) 创建图案，返回图案对象。

参数 repeat 表示图像如何重复，与 css 中 background-repeat 的取值相同。

可以把这个对象赋给 fillStyle 或 strokeStyle 属性，从而以渐变填充 或描画绘制的图形。

# 图像数据

调用上下文的 getImageData(x,y,width,height) 方法获取原始图像数据，返回一个imageData对象。

imageData对象包含三个属性，width、height 和 data。

data 为一个数组，每四个为一组表示一个像素信息，分别是 rgba(red,yellow,blue,alpha) 中代表红黄蓝透明度。每个值都在0~255中，连alpha都是。

# 合成

上下文有两个属性，globalAlpha 和 globalCompositionOperation。

globalAlpha 是指全局透明度，范围为0~1，分别表示不透明到透明。

设置了globalAlpha，可使图案透过下面图案，达到合成的效果，但真实的合成是使用另一个属性。

globalCompositionOperation 属性使新绘制的形状与上下文中已有的形状融合，可以取下列值：

  • source-over
    默认值，新图形覆盖原有图形。
  • source-in
    取交集，画布上只绘制出新图形与原有图形重叠的部分，其余部分全部透明。
  • source-out
    取+集，画布上只绘制出新图形不与原有图形重叠的部分，其余部分全部透明。
  • source-atop
    新图形只绘制出与原有图形重叠的部分，原有图形不受影响。
  • destination-over
    新图形绘制在原有图形下面，重叠部分只有原图形透明像素下的部分可见。
  • destination-in
    新图形绘制在原有图形下面，画布上只剩下二者重叠的部分，其余部分完全 透明。
  • destination-out
    新图形与原有图形重叠的部分完全透明，原图形其余部分不受影响。
  • destination-atop
    新图形绘制在原有图形下面，原有图形与新图形不重叠的部分完全透明。
  • lighter
    新图形与原有图形重叠部分的像素值相加，使该部分变亮。
  • copy
    新图形将擦除并完全取代原有图形。
  • xor
    新图形与原有图形重叠部分的像素执行“异或”计算。

# WebGL

是为 3D 画图，基于 OpenGL 着色器语言（GLSL）绘画。而 2d 的绘画是基于 js 语言的。

调用 canvas 的 getContext('webgl',options) 生成 3d 上下文 gl 时，options 参数可以设置 webGL 的属性，有下列属性。

  • alpha
    布尔值，表示是否为上下文创建透明通道缓冲区，默认为 true。
  • depth
    布尔值，表示是否使用 16 位深缓冲区，默认为 true。
  • tencil
    布尔值，表示是否使用 8 位模板缓冲区，默认为 false。
  • ntialias
    布尔值，表示是否使用默认机制执行抗锯齿操作，默认为 true。
  • premultipliedAlpha
    布尔值，表示绘图缓冲区是否预乘透明度值，默认为 true。
  • reserveDrawingBuffer
    布尔值，表示绘图完成后是否保留绘图缓冲区，默认为 false。 建议在充分了解这个选项的作用后再自行修改，因为这可能会影响性能。

常使用 try-catch 检测浏览器是否能创建 webGl 上下文。

if (drawing.getContext) { 
     try { 
     gl = drawing.getContext("webgl"); 
     } catch (ex) { 
     // 什么也不做
     }
     if (gl) { 
     // 使用 WebGL 
     } else { 
     alert("WebGL context could not be created."); 
     } 
}

# WebGL 和 OpenGL

WebGL 是 OpenGL ES 2.0 的 Web 版，所以内容皆是其衍生，即 JavaScript 所实现的 OpenGL。

  • 常量 WebGL 支持大部分 OpenGL 常量，但变量名可能会改变，即不包含 'GL_' 前缀。
