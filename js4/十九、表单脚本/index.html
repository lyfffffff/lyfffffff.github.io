# 十九、表单脚本

表单表现为一个 form 标签，里面包含许多诸如 input 的表单控件，可以进行提交。

# 表单基础

form 标签是 HTMLFormElement 类型，继承自 HTMLElement 类型，故他有普通标签的属性，也有自己的属性。

  • acceptCharset
    服务器可以接收的字符集，等价于 HTML 的 accept-charset 属性。
  • action
    请求的 URL，等价于 HTML 的 action 属性。
  • elements
    表单中所有控件的 HTMLCollection。
  • enctype
    请求的编码类型，等价于 HTML 的 enctype 属性。
  • length
    表单中控件的数量。
  • method
    HTTP 请求的方法类型，通常是"get"或"post"，等价于 HTML 的 method 属性。
  • name
    表单的名字，等价于 HTML 的 name 属性。
  • reset()
    把表单字段重置为各自的默认值。
  • submit()
    提交表单。
  • target
    用于发送请求和接收响应的窗口的名字，等价于 HTML 的 target 属性。

# 获取表单元素

可以直接使用 document.getElementBy[Id/Name/TagName] 获取，也可以使用 document.forms 获取所有表单元素

// 取得页面中的第一个表单
let firstForm = document.forms[0]; 
// 取得 name 属性为"form2"的表单
let myForm = document.forms["form2"];

# 提交表单

input|button[type='submit'] 和 input[type='image'][src=xxx] 都表现为一个按钮，可以进行提交表单。

当有提交按钮，且表单控件处于焦点时，按回车键也会触发提交表单。

textarea 控件例外，按回车键会换行。且没有提交按钮的表单在按回车键时不会提交。

表单在向服务器发送请求之前触发 submit 事件，可用于取消或验证表单。

<!-- 通用提交按钮 --> 
<input type="submit" value="Submit Form"> 
<!-- 自定义提交按钮 --> 
<button type="submit">Submit Form</button> 
<!-- 图片按钮 --> 
<input type="image" src="graphic.gif">

# 重置表单

input|button[type='reset'] 都表现为一个按钮，可以进行重置表单。

# 表单字段

表单字段的属性

  • disabled
    布尔值，表示表单字段是否禁用。
  • form
    指针，指向表单字段所属的表单。这个属性是只读的。
  • name
    字符串，这个字段的名字。
  • readOnly
    布尔值，表示这个字段是否只读。
  • tabIndex
    数值，表示这个字段在按 Tab 键时的切换顺序。
  • type
    字符串，表示字段类型，如"checkbox"、"radio"等。
  • value
    要提交给服务器的字段值。对文件输入字段来说，这个属性是只读的，仅包含计算机上 某个文件的路径。
# 表单字段的公共方法

focus 和 blur。

focus() 方法把浏览器焦点设置到表单字段，这意味着该字段会变成活动字段并可以响应键盘事件。

focus() 的反向操作是 blur()，其用于从元素上移除焦点

# 文本框

文本框为 input 和 textarea。

# 选择文本

文本框有 select() 方法，用于选中文本框的文本。且当选中文本框的文本时，会触发 select 事件。

# 获取所选文本

文本框有额外的属性 selectionStart 和 selectionEnd，用于截取选中的文本。

text.value.substring(text.selectionStart, text.selectionEnd);

# 获取部分文本

setSelectionRange(start,end) 从下标 0 开始，不含 end 下标，会触发 select 事件，选中部分文本。

# 输入过滤

当某些情况时，我们希望用户按下键盘时，输入框不响应。

  • keypress 即在按下键盘时，在 keypress 事件中进行处理，调用 event.preventDefault() 使得有些输入不做响应。

  • 剪切板
    即有复制、剪切、黏贴时触发的事件：

    • beforecopy：复制操作发生前触发，无法阻止 copy 事件。
    • copy：复制操作发生时触发。
    • beforecut：剪切操作发生前触发，无法阻止 cut 事件。
    • cut：剪切操作发生时触发。
    • beforepaste：粘贴操作发生前触发，无法阻止 paste 事件。
    • paste：粘贴操作发生时触发。

还可以直接获取剪贴板对象 clipboardData，通过 getData(MIME)、setData(MIME,text:String) 和 clearData 设置剪贴板的内容。

# 自动切换

在某些情况时，我们希望在输入足够长的文本后，自动切换表单控件。

在每个文本框输入到最大允许字符数时，自动把焦点切换到下一个文本框。

# 表单约束

表单控件添加了属性使其更多约束，在某些约束不满足的情况下，无法提交表单。

  • required 表示必填，否则无法提交表单。

  • 更多输入类型 即 type 新增了 mail 和 url

# 选择框编辑

即 select 和 option 标签，拥有表单标签的属性和方法，额外还有自己的方法和属性：

  • add(newOpt,relOpt) 添加新的 option，relOpt 表示在参考选项，新选项插在 relOpt 之前
  • multiple 多选
  • options 选择表单的所有选项
  • remove(index) 移除某个 index
  • selectedIndex 选中 option 的索引，若无选中则为 -1，若是多选的，也只会返回选中的第一项的索引，修改时移除所有已选项。
  • size 显示几行，超过出现滚动条
  • type 属性可能是 "select-one" 或 "select-multiple"

option 标签也有几个属性：

  • index 在众多 option 的排序
  • label 标签
  • selected 是否选中了当前选项，多选时不影响其他的选项
  • text 选项的文本。
  • value 选项的值
let text = selectbox.options[0].text; // 选项文本
let selectedOption = selectbox.options[selectbox.selectedIndex]; // 获取单项选择框的选项
console.log(`Selected index: ${selectedOption.index}\n` + 
 `Selected text: ${selectedOption.text}\n` + 
 `Selected value: ${selectedOption.value}`);

# 表单序列化

结合表单字段的 type 属性连同其 name 属性和 value 属性来进行序 列化

function serialize(form) {
    let parts = [];
    let optValue;
    for (let field of form.elements) { // 表单中的所有项
        switch (field.type) {
            case 'select-one':
            case 'select-multiple':
                if (field.name.length) { // 有 name 属性
                    for (let option of field.options) {
                        if (option.selected) {
                            if (option.hasAttribute) { // 若 option 有 value 属性则选择 value 否则是 text
                                optValue = option.hasAttribute('value') ? option.value : option.text;
                            } else {
                                optValue = option.attributes['value'].specified ? option.value : option.text;
                            }
                            parts.push(encodeURIComponent(field.name) + '=' + encodeURIComponent(optValue));
                        }
                    }
                }
                break;
            case undefined: // 字段集
            case 'file': // 文件输入
            case 'submit': // 提交按钮
            case 'reset': // 重置按钮
            case 'button': // 自定义按钮
                break; // 这些直接忽略
            case 'radio': // 单选按钮
            case 'checkbox': // 复选框
                if (!field.checked) {
                    break;
                }
                break;
            default:
                // 不包含没有名字的表单字段
                if (field.name.length) {
                    parts.push(`${encodeURIComponent(field.name)}=` + `${encodeURIComponent(field.value)}`);
                }
        }
    }
    return parts.join('&');
}

# 富文本编辑

PS：富文本应该使用富文本编辑器的插件，本节只当了解。

插入 iframe 元素，在文档 onload 事件中，将 iframe 的 designMode 属性设为 'on'，就可以在页面进行输入了。

本质上是插入了一个完整的 html 文档，修改的是 body 元素。

designMode 的取值为 'on' 和 'off'，默认为 'off'。

<iframe name="richedit" style="height: 100px; width: 100px"></iframe> 
<script> 
 window.addEventListener("load", () => { 
 frames["richedit"].document.designMode = "on"; 
 }); 
</script>

# 设置 contenteditable 属性

"true"表示开启，"false"表示关闭，"inherit"表示继承父元素的设置。

任何元素都可以设置。

# 与富文本交互

即使用 JS 将富文本变得比较丰富，例如文本加粗，用户做不到。

常使用 document.execCommand()

参数为要执行的命令、表示浏览器是否为命令提供用户界面的布尔值、和执行命令必需的值。

第二个参数一般是为 false。
