# 二十、JavaScript API

# 4，File 和 Blod

File 常在文件表单上获取，若是拖放文件则放置在 event.dataTransfer.files[n] 中，拥有以下属性

  • size
  • name
  • type MIME 类型
  • lastModifiedDate 上次修改时间

FileReader 用于读取文件，因为光使用 File 对象，是无法获取文件内容的。读取是异步的，progress、error 和 load，分别表示还有更多数据、发生了错误和读取完成，最终获取结果在 result 属性中。

  • FileReader.prototype.readAsText(file)
  • FileReader.prototype.redeAsDataURL(file)
  • FileReader.prototype.readAsArrayBuffer(file)
let reader = new FileReader()
let res; 
reader.reddAsArrayBuffer(file)
reader.onload = ()=>{
    res = reader.result
}

FileReaderSync 是读取文件的同步版本。只有在整个文件都加载到内存之后才会继续执行。慎用哦

Blod 则表示二进制对象，是 File 的父类。

  • new Blod(data,options) options 参数，并在其中指定 MIME 类型. Blod 还有一个 slice 方法，用于切割文件。
  • Blod.prototype.slice(startByte,endByte)

利用 Blod 创建 URL 地址 利用 window.URL.createObjectURL(params:Blod|File) 创建 URL，必须填入参数，可以传入 File 或 Blob 对象。这个函数返回的值是一个指向内存中地址的字符串。

# 5，媒体元素

媒体元素指的是 html5 新增的录音和视频标签，分别为 <audio> 和 <video>。 和 image 标签一样，使用 src 属性来内嵌外部音频和视频。

<!-- 嵌入视频 --> 
<video src="conference.mpg" id="myVideo">Video player not available.</video> 
<!-- 嵌入音频 --> 
<audio src="song.mp3" id="myAudio">Audio player not available.</audio>

此外，由于不同浏览器支持的视频和音频格式不同，可以使用一个或多个 <source> 元素代替，去掉 src 属性。

<!-- 嵌入视频 --> 
<video id="myVideo"> 
 <source src="conference.webm" type="video/webm; codecs='vp8, vorbis'"> 
 <source src="conference.ogv" type="video/ogg; codecs='theora, vorbis'"> 
 <source src="conference.mpg"> 
 Video player not available. 
</video> 
<!-- 嵌入音频 --> 
<audio id="myAudio"> 
 <source src="song.ogg" type="audio/ogg"> 
 <source src="song.mp3" type="audio/mpeg"> 
</audio>
