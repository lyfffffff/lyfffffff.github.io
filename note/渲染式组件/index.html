# 渲染式组件

包含 render、JSX、函数式组件。

# render

render: (h) => h( nodeName: String, options?: Object, childrenNode?: String|Array<[h(),...]> )

返回一个 Vnode 是因为组件需要一个根节点。

# options 的选项

{
    class,
    style,
    attrs, // HTML attribute
    props,
    domProps, // DOM 属性
    on:{}, // 事件监听，修饰符使用前缀替换
    nativeOn, // 原生事件监听
    directives,
    scopedSlots:{name:props=>h('span',props.name)}, // 作用域插槽，子组件所传的数据被封装到 props 中。
    slot,
    key,
    ref,
}

Tips：HTML attribute 和 DOM prototype 的区别 一个是在 html 标签上定义的属性集合，一个是通过 querySelect/getElement 获取 html 的 DOM 对象的属性集合。 DOM 对象可以直接获取 html 上的常规特性（id，name 等）。 对于不常规的属性，需使用 element.getAttribute() 获取，但是不一定是一一对应的。 例如行内样式 style，DOM 对象获取的是 CSSStyleDeclaration 对象，getAttribute 获取的是样式字符串。

id.getAttribute('style') // 'color:red'
id.style // CSSStyleDeclaration
特性 HTML Attribute DOM property
值 为 null 和字符串 任意类型
大小写 不区分 严格区分
未定义 返回 null 返回 undefined
value 属性相互更新 attribute 修改，property 修改 property 修改，attribute 不修改

# 插槽

子组件使用 render 函数渲染，插槽使用 this.$slots 和 this.$scopedSlots 获取。

render(h) {
  return h('div', this.$slots.default) // 渲染为 `<div><slot></slot></div>`
}

render(h) {
  return h('div', this.$scopedSlots.default({text:'i am son'})) // 渲染为 `<div><slot text="i am son"></slot></div>`
}

父组件使用 render 函数渲染，在 h 函数的 options 中使用 scopedSlots 渲染，props 为子组件所传的作用域。

render(h){
  return h('div',[ h( child, { scopedSlots: { default: props => h('span','i am farther give')}})]) 
}

# JSX

render 一个一个写标签太繁琐了，故推出了 JSX，在 vue 中内置了 JSX，可以直接使用。

且 render 自行判断写的是何种渲染函数。

在 JSX 中当遇到 <，JSX 就当 HTML 解析，遇到 { 就当 JavaScript 解析。

JSX 很像 vue，但是有几点不同：

事件绑定变为驼峰写法，变量使用 { + this 绑定，例如 v-model/v-on="num" -> vModel/vOn={this.num}，

可以直接在标签上写表达式，

render 函数返回一个 html 模板，对于标签、文本等基础内容直接使用。

# 函数式组件

有些组件，仅用于展示和外部定义，由调用时所绑定的事件和数据定义细节，没有 data，this，生命周期，watch、computed、methods，称为函数式组件。

函数式组件有两种写法，一是在 template 中添加 functional，

一是在 script 添加 functional 属性为 true，且使用 render 渲染组件。

函数式组件下的 render 函数，除了参数 h 外，还多了一个 context，表示本组件的上下文，即指代 this。

可以使用 JSX，也可以使用 h 函数渲染。

context:{
    props, // props 对象
    chilren, // 子节点数组，表示使用该组件时，插入组件的元素 [VNode,...]，值为 this.$slots.default + 非作用域插槽
    slots, // 函数，调用返回插槽，当然也无法获取作用域插槽，和子节点数组很像，不过是有命名的对象 {default:[VNode,VNode],other:[VNode]}
    scopedSlots, // 等同于 render 中的 this.$scopedSlots 
    data, // 所有数据
    parent, // 父组件
    listeners, // 父组件监听的事件
    injections, // 在此组件 inject 的数据
}

函数式组件的使用

// far.vue
<Functional text="functional text">
    <div>1111</div>
    <div slot='test' scoped-name="son">{{son.name}} is this</div>
    <div>2222</div>
</Functional>

// functional.vue
export default {
    functional: true,
    props: {
        text: {
            type: String,
        },
    },
    inject: ["name"],
    /**
     * 渲染函数
     * @param {*} h
     * @param {*} context 函数式组件没有this, props, slots 等都在 context 上
     */
    render(h, context) {
        const { props, children, listeners, slots, scopedSlots, data } = context;
        let slotList = slots(); // 所有插槽集合
        return h(
            "div",
            {
                on: {
                    click: listeners.click, // 父组件监听事件 click
                },
            },
            [props.text, 
            slotList.default,
            scopedSlots.test({name:'testName'}),
            children]
        );
    },
};

// 最后渲染为
<div>
    functional text
    <div>1111</div>
    <div>2222</div>
    <div>testName is this</div>
    <div>1111</div>
    <div>2222</div>
</div>
