# vue 组件中传数据的方式

因为根组件 App，其余组件都是根组件后代，所以所有组件哪怕离得再远，也是有联系的。

只是联系远近会影响产生联系或发生交互的方式。

# props 和 emit

父组件在子组件上使用 v-bind 绑定属性，会被传入子组件，子组件使用 props 属性接收。

若子组件没有对父组件传来的值使用 props 接收，子组件会将该属性作为 dom 属性挂载在根元素上，并可在控制台查看。

可以在子组件上设置 inheritAttrs: fasle 取消显示。

此外，所有未被接收的属性被封装到一个 this.$attrs 对象中保存。

// son.vue
export default {
    inheritAttrs: false, // 是否在 dom 属性中
    methods:{
        show(){
            console.log(this.$attrs) // {a:1}
        }
    },
};

// farther.vue
<Son :a="a"></Son>

子组件不得擅自修改父组件传来的属性，只能通过 emit 发送事件给父组件，通知父组件修改、或进行其他操作。

对于子组件发送的事件，父组件通过 v-on 在子组件上监听。

在子组件中可以使用 this.$listeners 获取所有发送且被父组件监听的事件。

// son.vue
export default {
    inheritAttrs: false, // 是否在 dom 属性中
    methods:{
        show(){
            this.$emit("changeA")
            this.$emit("changeB")
            this.$listeners//{changeA}
        }
    },
};

// farther.vue
<Son @changeA="change"></Son>

# 父子组件双向绑定

即父子组件任一修改某属性，在父子组件皆会响应。

规定子组件发送 attributeName.update 事件，父组件监听并修改该属性。

// son.vue
this.emit("a.update",this.a)

// farthr.vue
@a.update="($event)=>a=$event"

此外，在 vue 修饰符中，对 v-model 有 sync 修饰符，可以用于父子组件双向绑定。

v-model.sync="a"

# this.$parent 、this.$children 和 this.$refs

返回自身的父组件和子组件，子组件为一个数组。

因为 this.$children 获取的是数组，故获取某个子组件通过：this.$children[inx]。

但是这就和 v-for 一样，没有给子组件添加标识的话，若子组件中进行增删改查，这时候 inx 获取的子组件与预期不符。

此外还可以在子组件上使用 ref 属性标识，使用 this.$refs 获取子组件，这个类似 JS 中的 document.querySelector('xxx') ，可以在 vue 中获取元素/匹配组件。

  • 子组件获取父组件数据
this.$parent.a = 1
this.$parent.showA()
  • 父组件获取子组件数据
this.$refs.child1.a = 2
this.$refs.child1.showA()

# 路由参数

路由可以使得页面动态渲染组件，可以通过路由向所渲染的组件传参。

设置一个动态参数路由：

// router/index.js
{
    name:'newPath',
    path: "/newPath/:name/:age", // 动态参数为 name 和 age
    component: testRouter,
    props: true, // 表示动态参数作为 props 值传入组件 testRouter
},

# props

若设置路由时，将 props 设为 true，则 route.params 被设为该路由组件 props 所接收的参数。

# 传递和接收参数

<router-link to="/newPath/lyffff/11111111">去去去</router-link>
<router-view></router-view>

// newPath.vue
props:{
    name:String,
    age:String
}
this.$route.params // {name:'lyffff',age:'11111111'}

使用 $router 修改路由，其中 path 和 query 搭配，name 和 params 搭配。

query 类似于 get，params 类似于 post。

this.$router.push({path:'/newPath',query:{name:'wang',age:2}}); // 地址栏为 /newPath?name=wang&age=2
this.$router.push({name:'newPath',params:{name:'wang',age:2}}); // 地址栏为 /newPath/wang/2

// path 和 params 使用会不显示 params
this.$router.push({path:'newPath',params:{name:'wang',age:2}}); // 地址栏为 /newPath

# 事件总线——eventBus

即创建一个变量，每个组件都能使用，在某个组件发送一个事件，某个组件接收并完成，就和公交车一样，随意上下。

与全局 axios 一样，可以挂载在 vue 的原型中，每次使用 vue 中，都可以在 this.$bus 中找到它。

// main.js
Vue.prototype.$bus = new Vue()
this.$bus.$emit(event, this.event); // 触发事件（事件名，参数）

this.$bus.$on(event, (data) => {
	console.log('event', data)
); // 监听事件

this.$bus.$off(event) // 移除事件

this.$bus.$off(event).$on(event, () => {
	this.show = true;
}) // 若事件先前存在，赋值前先将其删除

# provide 和 inject

provide/inject 中为称为提供和注入，用于直系组件间传递数据，直系指的是父、爷、祖爷等关系，也就是你祖先给你留下的东西。

# provide:Function:()=>{} | Object

若 provide 为函数，需返回一个对象，此对象中的属性作为提供给后代的资源。

# inject:Array<String> | Object< {key : string|symbol|object} >

若 inject 为数组，则其中的项为字符串，表示注入祖先的所提供的属性名。

若 inject 为对象，对于每一个属性，皆包含 from 和 default 属性。

from 表示在可用的注入内容中搜索用的 key，default 当然就是默认值。

provide(){ return { name:this.name } }
inject:['name']
inject:{
    myName:{
        from:'name',
        default:'lyf' // 若没有搜索到祖先的 name 属性，自身 myName 的 value 值为 'lyf'
    }
}

provide 和 inject 并不是响应式，当祖先 provide 一个 data 数据时，在祖先组件中修改，后代组件无法接收到。在后代组件中修改，祖先组件也无法响应。

# 子组件响应式获取 provide

若想在后代组件中实时响应祖先组件所 provide 的数据。

需将祖先组件的数据以函数的形式获取，当数据变化时，函数的调用结果也不同。

在后代组件中使用 computed 不断监听函数的返回值，若有变化时，使用watch接收

// 祖先
provide(){ return {showName:()=>this.name} }
// 后代
computed(){}
