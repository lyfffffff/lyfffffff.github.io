# 客户端存储

网站分为静态网站（static sites）和动态网站（dynamic sites）。

静和动的意思是，是否根据使用的用户不同，页面呈现不同的效果，当然布局是差不多的，区别是内容。

动态网站为了方便获取变化的数据，会在服务端存储数据，使用 API 获取后将数据插入静态页面模板中生成 HTML 渲染到浏览器上。

随着 PWA 的兴起，客户端存储变的至关重要，与服务器存储互补，即页面变化但数据想保留，故把数据存储在浏览器上，可以在需要的时候获取。

对于 web 应用，有三种客户端存储：cookie、web storage、indexedDB。

可以通过谷歌开发者工具的 Application 模块查看页面的客户端存储。

# cookie

这是最早的客户端存储方式，它所存储的数据会过期，大小有限，个数有限，但 cookie 依然常见，因为 cookie 要求服务器在响应 HTTP 请求时，通过发送 Set-Cookie HTTP 头部包含会话信息。

# cookie 字段

cookie 有 12 个字段

name value domain path Max-age size httpOnly secure samesite sameParty partition key priority
名称 值 域 路径 过期时间 大小 仅用于放置在 http 头部，即 document.cookie 无法获取 安全标识 同站 同站 同站，分域名的 key 优先级

# 优缺点

每次请求必须携带在请求头。

只能使用 document.cookie 操作。

# js-cookie 库

弥补原生 cookie 难以操作

import Cookies from 'js-cookie' 
Cookies.set(name, value , cofig)
Cookies.get(name) 
Cookies.remove(name, config)

# Web storage

Storage 所提供的实例包括 sessionStorage 和 localStorage。

当然，它所存储的数据也会过期，大小有限，个数有限

皆有 Storage 的方法：removeItme(name)、setItem(name,value)、getItem(name)、key()、clear(index)。

除了 IE8 以下，现在的浏览器都是同步存储 storage 数据。

当 Storage 实例发送变化时，即 sessionStorage 和 localStorage 添加/删除/设置值时，会触发 window 的 storage 事件，事件包含 domain、key、newValue、oldValue 等属性。

window.addEventListener("storage",
 (event) => alert(event.domain));

# sessionStorage

储存会话数据，即浏览器关闭后，数据消失，且只能在存储该数据的页面访问。

# localStorage

比 sessionStorage 持久，要访问同一个 localStorage 对象，页面必须来自同一个域（子域不可以）、在相同的端 口上使用相同的协议。

# indexedDB

即为数据库存储，前身是 web sql，但 html5 已废弃，改为 indexedDB，在前端页面创建数据库。

  • 初始化 indexedDB 1，调用 open，数据库名和版本请求数据库，若无该数据库就创建，此步为异步操作。版本应随着修改手动升级。 2，为 indexedDB 添加成功和失败的事件处理程序，并在事件的 event.target.result 中获取数据库
let db
let request = indexedDB.open(dbName:String,version:Number)
request.onsuccess = (event)=>{ db= event.target.result}
request.onerror = (event)=>{ db= event.target.result }
  • 创建表格 新建数据库或升级数据库，触发 onupgradeneeded 事件，通过 event.target.result 获取 db，调用 db 的 createObjectStore() 创建表并返回表实例，调用表实例的 createIndex 创建列。
db.onupgradeneeded = (event)=>{
    let db = event.target.result
    let objectStore = db.createObjectStore(tableName:String,option:Object<{keyPath:String}>)
    objectStore.createIndex(indexName:String,option:Object)
}
  • 数据库事务 调用数据库的 transaction 方法创建事务，根据参数获取单个或多个表格的事务。 事务调用 objectStore 进入到表格，表格实例有 add、put、get、delete、clear 等方法，调用这些方法返回的实例，也有 onerror 和 onsuccess 方法。 事务本身拥有 oncomplete 和 onerror 方法。
let transaction = db.transaction(store?:String|Array,type:'readOnly'|'readwrite'|'versionchange')
let store = transaction.objectStore(store)
store.get('lyf')

特性 cookie localStorage sessionStorage indexedDB
大小 4k 5M 5M 不限制
