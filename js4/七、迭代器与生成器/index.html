# 七、迭代器与生成器

# 7.1，迭代器 Iterator

迭代器是一种访问数据中的元素的工具，可以说是为 for...of 服务的，因为只有拥有迭代器的数据，才能使用 for...of，否则报错。

通过调用迭代器 Iterator 的 next 方法获取数据的元素，且每调用一次 next，迭代器都会指向数据的下一个元素。

next 返回一个迭代结果（IteratorResult）对象 obj = {done:, value:}；

其中 done 为 Boolean 值，表示是否迭代完成，遍历到数据末尾时为 true；

value 表示元素的值，当 done 为 true 时 value 为 undefined。

# 7.1.1，可迭代对象

具备 Iterable 接口的称为可迭代对象。

可迭代对象有 Array、Map、Set、String、TypedArray，函数 arguments 对象和 NodeList 对象。

对于没有迭代器的 js 结构，使用迭代器方法，会报错。

这些对象元素有限且具有无歧义的遍历顺序；

# Symbol.iterator

可迭代对象有一个内置属性 Symbol.iterator，类型为函数，调用返回一个迭代器 Iterator；

迭代器本身也有 Symbol.iterator 属性，调用时返回自身。

# 7.1.2，迭代器使用

一般不会使用 arr[Symbol.iterator]().next()，而是在某些方法内部调用迭代器，单独迭代器有种将遍历可视化的感觉。

迭代器有种将遍历可视化的感觉一样。

例如 for...of、数组解构、扩展运算符、Array.from()、Promise.all()、Promise.race()、yield*。

  • 操作顺序 拥有 Iterable 接口 -> 可迭代对象 -> 调用 Symbol.iterator 属性 -> 返回迭代器 -> 调用迭代器的 next 方法 -> 返回迭代结果对象
let arr = ['a', 'b', 'c'];
let iter = arr[Symbol.iterator](); // 执行迭代器属性函数，返回迭代器
iter.next() // 调用迭代器的 next 返回迭代器结果 {value:'a', done:false}
iter.next() // {value: 'b', done:false}
iter.next() // {value:'c', done:false}
iter.next() // {value:undefined，done:true} 接下来调用 next()都返回此

iter[Symbol.iterator]() == iter // true
  • 修改迭代器 为新的类添加 [Symbol.iterator] 构造器，就可以使用 for...of 遍历。

为了让迭代器包含 next 方法，有两种思路。

迭代器返回一个带有 next 方法的对象。

或将 next 方法放在类上，迭代器只需返回类本身的this。

若 next 在类上，对于同一个实例来说，每次遍历返回的迭代器的 next 都是同一个，存在缺陷。

为了让每次 for...of 都是新的遍历，迭代器返回一个对象。

class newArray{
    constructor(stop){
        this.data = [1,2,3,4,5,6]
        this.stop = stop;
    }
    [Symbol.iterator](){ // 返回 this
    let stop = this.stop
        return {
            limit:1,
            next(){ // 在调用迭代器时使用
            if(this.limit <= stop){
                return {done : false,value : this.limit++}
            }
            return {done : true,value : undefined}
    }
        }
    }
}
let arr = new newArray(3)
for (var value of arr) { // [Symbol.iterator] -> next
  console.log(value); // '1' '2' '3'
}

# 7.2，生成器 Generator

以上得到迭代器有两种方式，一是从存在迭代器的数据结构中获取，一是构造新类然后设置迭代器。

都有种为了儿子生个妈的感觉，故生成器 Generator 的作用就是跳过构造函数专门生成迭代器的。

生成器表现为一个函数，为了区分生成器和普通函数，生成器函数名一般带星号 function *test(){}。

不可以使用 new 实例化生成器，但是实例继承生成器函数上的原型对象。

调用生成器函数返回迭代器实例，并不执行函数内部代码。

当调用返回的迭代器的 next 时，才执行函数内部代码。

返回迭代器结果对象 { done,value }，value 是生成器函数的返回值，默认是 undefined，可以在生成器中修改。

function *generatorFn(){
}
const g = generatorFn()
g.next() // {value:undefined，done:true}
g[Symbol.iterator]() === g // 本身就是一个迭代器

// 自定义 return
function *generatorFn(){
    return 'xxxx'
}
const g = generatorFn()
g.next() // {value:'xxxx', done:true}
  • 修改数据的迭代器 在需要添加迭代器的 js 结构，将生成器添加到 js 结构的 Symbol.iterator 属性上。
let obj = {name:'', age:18}
obj[Symbol.iterator] = function* test() {
    let keys = Object.keys(this)
    for(let key of keys){
        yield [key，this[propKey]]
    }
}

# 7.2.1，yield 关键词

yield 必须在生成器函数中使用，且不能在嵌套在非生成器函数里。

控制生成器开启和暂停，感觉有点像 async 和 await。

在遇到此关键词前，函数内部正常执行，遇到时执行停止，需再次调用迭代器 next() 激活接下代码。

yield 后面可以跟一个表达式，表示 next() 至此的返回值，会先计算表达式再返回。

// yield 嵌套在非生成器函数里
function *test(){
    function test(){
        yield '' // error
    }
}

// yeild 的功能
function *generatorFn(){
    console.log('xxxx')
    yield 1+1       // 第一次 next() 至此，先计算表达式
    console.log('ssss')
    yield 'ssss'       // 第二次 next() 至此
    return 'finally'   // 执行至此，退出
}
const g = generatorFn()
g.next() // 'xxxx' {value:2, done:false}
g.next() // 'ssss' {value:'ssss', done:false}
g.next() // {value:'finally', done:true}

yield 可以充当 next 的参数，因为第一次调用 next() 时执行到 yield 之前，传参无法接收。

故第 n 个 yield 存储着 第 n + 1 次调用 next() 时所传的参数。

yield 作为变量，可以和其他表达示一起使用，哪怕 yield 后不接表达式，都需要用小括号圈起来表示。

若 return yield xx，则倒数第二次调用 next 返回 'xx'，最后一次调用 next 返回值是 yeild ，即 next 的参数。

// yield 在其他表达式上使用
function *generatorFn(){
    console.log('xxx' + (yield 1)) 
}
const g = generatorFn()
g.next() // '1'
g.next(10) // 'xxx10'

// return + yeild + 表达式
function *test(){
    return yield 'test'
}
t.next() // 执行至 yield，返回 {value:'test', done:false }
t.next('change') // 将 'change' 传给 yield，变成 return 'changhe'，返回 {value:'change', done:true}

// yeild 在表达式里
function * f(x){
    let y = yield (x + 1) // yield 3 -> yield
    return y
}
let g = f(2)
g.next() // {value : 3, done: false}
g.next() // {value : undefined， done: true}，因为 yield 为 undefined

# 7.2.2，Generator.prototype.throw(?param)

生成器函数返回的迭代器，继承生成器原型上的 throw 方法。

调用 throw 时，在函数执行的位置，即 yeild 关键词处抛出错误。

必须在至少执行一次 next 后调用 throw，否则报错。

对于错误需使用 try...catch 捕获，且 throw 的参数为 catch 的参数。

若 try...ctach 在生成器函数内部，捕获以后，会附带执行下一条 yield 表达式，也就是说，相当于多附带执行一次 next 方法，并不影响下一次遍历。

若在函数外部使用 try...catch 包裹，则迭代器认为这是停止信号，再次调用 next 时，返回 {done,value}。

var g = function* () {
  try {
    yield console.log("我是第 0 个");
    yield console.log("我是第一个");
    yield console.log("我是第 x 个");
  } catch (e) {
    console.log("内部捕获", e);
  }
  yield console.log("我是第二个");
  yield console.log("我是第三个");
  yield console.log("我是第四个");
  yield console.log("我是第五个");
  yield console.log("我是第六个");
};

var i = g();
i.next(); // '我是第 0 个'
i.throw("第一次出错"); // '内部捕获''第一次出错''我是第二个'
i.next(); // '我是第三个'

try {
  i.throw("第二次出错");
} catch (e) {
  console.log("外部捕获", e);
} // '外部捕获', '第二次出错'
i.next(); // 直接返回 value-done 对象

# 7.2.3，Generator.prototype.return(?param)

生成器函数返回的迭代器，继承生成器原型上的 return 方法。

return 相当于直接忽略 yield，直接抵达函数末尾，可以有一个参数 param，表示 value 的值，done 也是 true。

# 7.2.4，生成器方法总结

其实 next、throw、return 都是操作关键字 yield。

next 表示将 yield 替换成方法参数。

throw 表示将 yield 替换成 throw 语句，直接到 catch 后。

return 忽略 yield 和 return 语句，直接返回。

# 7.2.5，yield* 关键字

yield* 用于在生成器内部调用其他生成器。

生成器内部调用其他生成器，本质是，生成器函数执行至此时，迭代器内部包含一个迭代器，按照栈的思想，先执行内部的迭代器，执行完毕后，再执行外部迭代器剩余部分。

一是使用 for...of 手动遍历内部迭代器，二是使用 yield* 执行内部迭代器。

表现出来为，next 抵达 yield* 所指向迭代器时，会执行至 所指向迭代器 的 yield，并在 所指向迭代器 执行完毕之前，next 都在 所指向迭代器 内部。

function* f() {
  yield "xxx";
  yield "test";
}

function* f1() {
  yield "hello";
  yield* f(); // 调用生成器，并遍历迭代 类似于 for(let key of f()){ yield key}
  yield f(); // 返回迭代器 {value: f， done: false}
}
let iterator = f1();
iterator.next(); // {value:'hello'}
iterator.next(); // {value:'xxx'}
iterator.next(); // {value:'test'}
iterator.next(); // {value:f}
iterator.next(); // {value:undefined，done:true}

# 7.2.6，对象属性

生成器可以作为对象的属性

let obj = {
  *Generator() {},
};
let obj = {
  Generator: function* () {},
};
