# 前言

响应式原理是前端面试常考的一个知识点，毕竟响应式是 Vue 框架的核心，响应式并不是 v-model 双向绑定，v-model 只是响应式的应用。

当去查阅资料，得出：Vue 内部主要靠观察者劫持 Object.defineProperty() 的 getter 和 setter 方法，在每次 get 数据的时候注入依赖，在 set 数据的时候发布消息给订阅者，从而实现 DOM 的更新。

这里有几个概念：观察者，订阅者，注入依赖。DOM 更新还涉及到虚拟树等。

# 何为响应式

在学习 css 时，我们学过响应式布局，即页面的组件根据浏览器的变化，一套代码兼容所有浏览器。

而 Vue 的响应式指的是数据，当 data 中的数据变化时，用户所看到的 Dom 视图也发生变化。

那么问题来了，原生的 js 有没有响应式呢？

首先原生的 js 把数据放置在 DOM 中就很困难。

一般是通过赋值，例如 a 变量赋值给某个 input 的 value 中。

这种方式只是复制了一个 a 的副本给 input 框，此时修改 a，input 框不会有任何改变。

而 vue 中，将属性 a 绑定在 input 的 value 中，a 改变，input 的 value 也会改变。

# 是否 vue data 的所有属性都有响应式？

当然不是，在 data 中定义的属性，是响应式的，若在其余地方新增的属性，是不具备响应式的。

这是因为 Vue 在初始化时操作了 data 使其变成响应式的，后添加的属性不具备这层包装，故不是响应式的，但是可以使用 this.$set 定义属性使其变成响应式的。

# step 3 Dep 管理者

defindReactive 中 getter 调用了 dep.depend()，setter 调用了 dep.notify()，这就是依赖收集和触发更新的起点。

这里的 dep 是 Dep 的实例，所谓依赖收集，就是把一个数据用到的地方收集起来，在这个数据发生改变的时候，统一去通知各个地方做对应的操作。

故 dep 是的 depend 和 notify 是收集和通知，属于过渡者，而被通知并做出反应的是订阅者 Watcher。

在 depend 方法中的 target 是订阅者，notify 中的 subs[i].update 也是订阅者。

let uid = 0
class Dep {
  constructor() {
    this.id = uid++;
    this.subs = []; // 操作subs数组
  }

  addSub(sub) {
    this.subs.push(sub);
  }

  removeSub(sub) {
    remove(this.subs, sub);
  }

  depend() {
    if (Dep.target) {
      Dep.target.addDep(this); // target 即是当前活跃的订阅者 watcher，调用其方法 addDep
    }
  }

  notify() {
    const subs = this.subs.slice();
    subs.sort((a, b) => a.id - b.id); // 根据 id 升序
    for (let i = 0, l = subs.length; i < l; i++) {
      subs[i].update(); // 通知订阅者更新
    }
  }
}

# step 4 Watcher 订阅者

由此 Observer、Dep 都已经在 initState 中一脉实例化并联系了，响应式数据和依赖管理都准备好了。

那么 Watcher 呢，Watcher 并不在 _init 中定义，而是在 vm.$mount 挂载。

vm.$mount 中执行 mountComponent 方法，mountComponent 中实例化 Watcher 和 updateComponent，后者用于更新整个 DOM 树。

联系前面的 Dep，收集依赖是 getter 是通过 Dep.depend 调用 Watcher.addDep，将当前 Watcher 加入 Dep 维护的依赖列表 newDepIds 中。

依赖触发是 setter 是通过 Dep.notify 调用所有依赖的 Watcher.update，然后加入异步更新队列，依次调用 run ，run 调用 get，get 使用新 data 创造虚拟 DOM ，再对比局部更新整颗 DOM 树。

class Watcher {
 this.getter = expOrFn
 this.value = this.get() // 更新 DOM 视图
  get () {
    pushTarget(this) // 将 Watcher 挂载到 Dep 中的 target 中
    let value
    const vm = this.vm
    value = this.getter.call(vm, vm) // 调用了 updateComponent 更新视图，但没有返回值，故 value 为 undefined
    if (this.deep) { 
        traverse(value)
      }
      popTarget() // 移除 target
      this.cleanupDeps()
    return value
    
  }

  addDep (dep) {
    const id = dep.id
    if (!this.newDepIds.has(id)) { // 将 data 属性存储起来
      this.newDepIds.add(id)
      this.newDeps.push(dep)
      if (!this.depIds.has(id)) {
        dep.addSub(this)
      }
    }
  }

  cleanupDeps () {
    let i = this.deps.length
    while (i--) {
      const dep = this.deps[i]
      if (!this.newDepIds.has(dep.id)) {
        dep.removeSub(this)
      }
    }
  }

  update () { // 更新试图时调用
      this.run() || queueWatcher(this) // 异步使用后者，一般是异步，queueWatcher 先排队，后来还是调用 Watcher.run
  }

  run () {
    if (this.active) {
      const value = this.get() // 此处又会更新视图
  }
}
