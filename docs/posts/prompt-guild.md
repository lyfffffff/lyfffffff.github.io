# 🚀 前端工程师 Prompt 提升指南（实战版）

> 目标：让 AI 像高级工程师一样，帮助你开发、学习、Debug

---

# 🧠 一、核心认知（最重要）

## ✅ 1. Prompt 的本质

> ❌ 不是写长  
> ✅ 是信息清晰 + 指向明确

---

## ✅ 2. 不需要每次写模板

- 模板只用于：
  - 第一次提问
  - 信息复杂时

- 日常使用：
  👉 一行提问 + 多轮对话

---

## ✅ 3. 最重要能力

👉 **和 AI 对话，而不是一次性提问**

---

# ⚡ 二、一行高质量提问公式（核心）

````txt
【技术栈】+【场景/目标】+【问题】+【关键信息】+【你想要的结果】
~~~

------

## ✅ 示例

### 🧩 开发

​```txt
Vue3 + TypeScript 实现动态表单，需要字段联动和校验，数据结构如下xxx，请给完整代码和设计思路
````

### 🐞 Bug

```txt
Vue3 列表数据更新但视图没刷新，代码如下xxx，没有报错，请分析原因并给排查步骤
```

### ⚡ 性能

```txt
Vue3 渲染1000条列表卡顿，使用v-for，代码如下xxx，请分析瓶颈并优化
```

### 📚 学习

```txt
Node.js 事件循环是怎么工作的，在实际项目中有什么影响，请结合示例讲清楚
```

---

# 🧩 三、简化 Prompt（日常用）

## ✅ 通用版

```txt
问题：
【做什么】

技术栈：
【Vue3 / TS / Node 等】

代码：
【关键代码】

要求：
- 给可运行代码
- 符合最佳实践
- 说明关键点
```

---

## ✅ 快捷版本

### 🧩 写功能

```txt
帮我用 Vue3 实现：
【需求】

要求：完整代码 + 可维护 + 处理边界情况
```

---

### 🐞 查 Bug

```txt
问题：
【现象】

代码：
【代码】

报错：
【报错】

请给排查步骤 + 修复方案
```

---

### 📚 学习

```txt
帮我讲清楚：
【知识点】

要求：
- 通俗解释
- 原理 + 示例
- 实际应用
```

---

# 🔁 四、多轮对话正确姿势（关键）

## ✅ 标准流程

### 1️⃣ 高质量开局

描述清楚问题 + 给代码

---

### 2️⃣ AI 分析

AI 给多个可能原因

---

### 3️⃣ 你筛选（很关键）

```txt
我觉得第2个原因更可能，我们先排查这个
```

---

### 4️⃣ 实验 + 反馈

```txt
我试了你的方法：
- A 成功
- B 失败

现在结果是xxx
```

---

### 5️⃣ 深挖本质

```txt
为什么会这样？本质原因是什么？
```

---

### 6️⃣ 要最佳实践

```txt
实际项目中怎么避免？
```

---

# 🐞 五、Debug 实战套路（核心能力）

## 🎯 Debug = 5步法

1. 描述问题（像提 issue）
2. 列出可能原因
3. 聚焦一个原因
4. 验证 + 反馈
5. 定位根因 + 修复

---

## ✅ 标准 Debug Prompt

```txt
我用 Vue3 + TypeScript，

问题：
切换筛选后数据更新但页面不刷新

补充：
- 使用 Pinia
- 无报错

代码：
【代码】

请分析原因（按优先级）
```

---

## ✅ Debug 引导语（非常实用）

### 聚焦问题

```txt
我们先只看这个原因
```

### 要步骤

```txt
给我具体排查步骤
```

### 反馈结果

```txt
我试了，结果是xxx
```

### 深挖原理

```txt
为什么会这样？
```

### 要工程方案

```txt
怎么在项目中避免？
```

---

# 🧠 六、高级使用技巧

## ✅ 控制 AI 输出质量

```txt
请按照生产级代码标准输出
不要泛泛而谈，请具体到代码层面
```

---

## ✅ 提高工程实用性

```txt
请考虑真实业务场景，而不是 demo 实现
请考虑可维护性和扩展性
```

---

## ✅ 防止 AI 胡说

```txt
如果不确定，请明确说明
```

---

# ❌ 七、常见错误

## ❌ 问题太模糊

```txt
为什么不行？
```

---

## ❌ 不给代码

```txt
帮我看看代码
```

---

## ❌ 不反馈结果

👉 AI 无法继续推理

---

## ❌ 每轮都用模板

👉 会打断上下文

---

# ⭐ 八、核心总结

## 🎯 一句话

👉 模板是“启动器”，对话才是“主战场”

---

## 🎯 高级 vs 低级用法

| 级别    | 用法           |
| ------- | -------------- |
| ❌ 低级 | 让 AI 给答案   |
| ✅ 高级 | 让 AI 陪你推理 |

## 🎯 最终目标

👉 让 AI 成为你的：

- 高级前端同事
- Debug 搭档
- 技术导师
